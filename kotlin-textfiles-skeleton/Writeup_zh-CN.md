# Kotlin: 文本处理

> 难度评价：2/5

> **照 spec 做！！！照 spec 做！！！照 spec 做！！！照 spec 做！！！**

基本和 Kotlin 小曲 一套做法。直接抄 spec。出题人一般都会在 spec 里面很详细描述解题方式，仔细读的话，基本每道题都是送分题。只要你熟悉 Kotlin，能把描述的算法用 Kotlin 语言来实现，就能很轻易地做完。

1. 照 spec。
2. 唯一难点是寻找 index 对应的 block 计算 block Offset。计算完之后，可以轻松做。rebalance 实现方式可以先 合并后拆分，这样来做事儿最少。
3. 每份试卷都会有一题让你用 Java 来写。如果熟悉 java 语法，还有 java 和 kotlin 之间的区别与其互通方式，这也不算什么难题。
4. 刷 withLock。
5. 直接照 spec 实现。一行答案，纯送分。
6. 压轴题。刷过 ppt 和其他题目的就会熟悉 HashMap 哈希的实现方式。需要知道如何建立并实现匿名对象。你一深入理解哈希表的运行方式，马上能做完。如题实现。

> 综合评价：比较容易，其中第二题和第六题略难。可以当作练习来做。
